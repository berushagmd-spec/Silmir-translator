# Sil'mir Telegram Bot

Telegram-бот переводит русский текст на Sil'mir по словарю и простым грамматическим правилам.

Логика не требует нейросети: бот читает `.txt`-словарь из архива/папки, строит SQLite-индекс и переводит текст самым длинным совпадением. Если слово не найдено, оно остаётся в квадратных скобках, чтобы было видно, что его нужно добавить в словарь.

## Структура

```text
.
├── src/silmirbot/
│   ├── bot.py             # Telegram-бот на aiogram
│   ├── config.py          # настройки из .env
│   ├── dictionary.py      # чтение txt/zip и построение индекса
│   ├── normalize.py       # нормализация русского текста
│   ├── rules.py           # базовые правила Sil'mir
│   └── translator.py      # переводчик
├── data/
│   └── archive/           # сюда можно положить распакованную папку словаря
├── requirements.txt
├── .env.example
└── docker-compose.yml
```

## Как положить словарь

Есть два варианта.

Вариант 1 — положить zip:

```text
data/Сильмир.zip
```

Вариант 2 — распаковать архив так, чтобы внутри были `.txt` файлы, например:

```text
data/archive/словарр/Время и числа 001.txt
```

Бот сам обходит все `.txt` рекурсивно. Формат строки словаря должен быть таким:

```text
silmir - русский смысл
kair - камень
kairən - камень как прямой объект
```

## Запуск локально

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
pip install -e .
cp .env.example .env
```

В `.env` вставь токен от BotFather:

```env
BOT_TOKEN=твой_токен
SILMIR_DATA_PATHS=data/Сильмир.zip,data/archive,archive,словарр
SILMIR_INDEX_PATH=data/silmir.sqlite3
```

Первый запуск создаст индекс `data/silmir.sqlite3`:

```bash
python -m silmirbot.bot
```

Если обновил словарь, пересобери индекс:

```bash
python -m silmirbot.dictionary --rebuild
```

Или поставь в `.env`:

```env
SILMIR_REBUILD_INDEX=1
```

## Запуск через Docker

```bash
cp .env.example .env
# вставь BOT_TOKEN в .env
docker compose up -d --build
```

## Команды бота

`/start` — помощь.

`/translate текст` — перевести текст.

`/reload` — пересобрать индекс словаря без перезапуска бота.

Также можно просто отправить обычное сообщение, и бот переведёт его.

## Как работает перевод

1. Текст разбивается на слова и знаки препинания.
2. Переводчик ищет самые длинные фразы в словаре: сначала 7 слов, потом 6, 5 и так далее.
3. Если есть точное совпадение в словаре, берётся лучший вариант.
4. Если перед словом стоит `два` или `много`, бот пытается взять готовую форму из словаря.
5. После известных переходных глаголов бот пытается поставить существительное в прямой объект: например `камень` → `kairən`.
6. Частица `не` перед известным глаголом переводится как отдельное `na`, а для обычного слова бот пробует найти форму с приставкой `na`.

Пример:

```text
я не вижу камень
```

Результат:

```text
yaś na velims kairən
```

Переводчик намеренно оставляет неизвестные слова как `[слово]`, потому что Sil'mir — авторский язык, и лучше явно видеть пробелы словаря, чем получать выдуманный перевод.
